# День 4 — Baseline без обучения трансформера

## Что мы делаем и зачем

Сегодня мы впервые решаем реальную задачу — **классификацию тональности текста**
(sentiment analysis). Но делаем это без обучения самого трансформера.

Подход: берём предобученный DistilBERT "как есть", извлекаем из него эмбеддинги
(CLS-вектора), и обучаем на них простую логистическую регрессию.

Это называется **feature extraction** (извлечение признаков) — в отличие от
**fine-tuning** (дообучения), где мы меняем веса самого трансформера.

```
Feature extraction (сегодня):
  Текст → [DistilBERT замороженный] → CLS-вектор → [LogReg обучается] → класс

Fine-tuning (потом):
  Текст → [DistilBERT обучается] → CLS-вектор → [Классификатор обучается] → класс
```

## Зачем нужен baseline?

Baseline (базовая линия) — это простое решение, с которым мы потом сравниваем
более сложные подходы. Без baseline невозможно понять, насколько хорошо работает
fine-tuning — может, простая логрег и так даёт 95%?

Правила хорошего baseline:
1. Должен быть **простым** — чтобы можно было быстро запустить
2. Должен быть **разумным** — не случайное угадывание, а реальный подход
3. Должен быть **воспроизводимым** — `random_state=42` для повторяемости

## Датасет IMDB

IMDB — стандартный датасет для sentiment analysis. 50 000 отзывов на фильмы,
размеченных как positive (1) или negative (0):

- 25 000 для обучения, 25 000 для теста
- Сбалансированный: 50% positive, 50% negative
- Средняя длина отзыва: ~230 слов

Мы берём подвыборку из 2000 текстов для скорости (полный датасет на CPU
обрабатывался бы слишком долго).

```python
dataset = load_dataset("imdb", split="train[:2000]")
```

`split="train[:2000]"` — синтаксис Hugging Face Datasets для взятия первых
2000 примеров из обучающей выборки.

## Feature Extraction: пошагово

### Шаг 1: Токенизация

Та же функция, что и в День 1, но теперь мы используем `distilbert-base-uncased`
(английский, без учёта регистра) — для IMDB не нужна мультиязычность.

### Шаг 2: Извлечение CLS-эмбеддингов

```python
with torch.no_grad():
    outputs = model(**tokens)
cls = outputs.last_hidden_state[:, 0, :]
```

Что здесь происходит:
1. `model(**tokens)` — прогоняем токены через все 6 слоёв DistilBERT
2. `outputs.last_hidden_state` — тензор `[batch_size, seq_len, 768]`
3. `[:, 0, :]` — берём первый токен (CLS) для каждого текста в батче
4. Получаем матрицу `[batch_size, 768]` — один вектор на текст

### Почему батчами?

2000 текстов по 128 токенов × 768 измерений — это много данных. Если передать
всё сразу, может не хватить памяти. Поэтому обрабатываем по 16-32 текста:

```python
for i in range(0, len(texts), batch_size):
    batch_texts = texts[i : i + batch_size]
    ...
```

### Шаг 3: Logistic Regression

Логистическая регрессия — это линейный классификатор. Она ищет гиперплоскость
в 768-мерном пространстве, которая разделяет positive и negative отзывы.

```python
clf = LogisticRegression(max_iter=1000, n_jobs=-1)
clf.fit(X_train, y_train)
```

- `max_iter=1000` — максимум итераций оптимизации (по умолчанию 100, часто
  не хватает для высокоразмерных данных)
- `n_jobs=-1` — использовать все ядра CPU

## Метрики

### Classification Report

```
              precision    recall  f1-score   support
    negative       0.84      0.85      0.84       200
    positive       0.85      0.84      0.84       200
```

- **Precision** — из всех предсказанных positive, сколько реально positive?
- **Recall** — из всех реально positive, сколько мы нашли?
- **F1-score** — гармоническое среднее precision и recall
- **Support** — количество примеров в каждом классе

### Macro F1

`f1_score(y_test, y_pred, average='macro')` — среднее F1 по всем классам
без учёта их размера. Для сбалансированного датасета (50/50) macro F1 ≈
обычному accuracy.

## Почему CLS без fine-tuning работает неидеально?

DistilBERT обучался на задачах MLM (предсказание замаскированных слов) —
он понимает язык в целом, но не обучался конкретно на sentiment analysis.

Его CLS-вектор кодирует **общий смысл** предложения, но не оптимизирован
для разделения positive/negative. Fine-tuning (День 5+) исправит это —
мы подстроим все веса модели под нашу задачу, и F1 вырастет.
