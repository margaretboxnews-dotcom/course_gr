## Инференс (Inference)

**Инференс** — это использование обученной модели для предсказания на новых данных.
В отличие от обучения, при инференсе:

- Модель в режиме `eval()` (dropout выключен)
- Градиенты не вычисляются (`torch.no_grad()`)
- Данные проходят только forward pass (нет backward)

Для fine-tuned модели инференс выглядит так:

```
Текст → tokenizer → input_ids → model → logits → softmax → класс
```

## Logits и Softmax

Модель возвращает **logits** — "сырые" числа, не нормализованные в вероятности:

```
logits = [−1.5, 2.3]   # отрицательный, положительный
```

`softmax` превращает logits в вероятности (сумма = 1):

```
softmax([-1.5, 2.3]) = [0.023, 0.977]
→ 97.7% positive, 2.3% negative
```

Формула softmax:

```
softmax(xᵢ) = exp(xᵢ) / Σⱼ exp(xⱼ)
```

Для предсказания берём `argmax` — индекс максимального значения.

## Confusion Matrix

Confusion matrix показывает, сколько раз модель предсказала каждый класс
правильно и неправильно:

```
                  Предсказано
                  neg    pos
Настоящий neg  [ TN     FP ]
          pos  [ FN     TP ]
```

- **TP (True Positive)** — правильно предсказано positive
- **TN (True Negative)** — правильно предсказано negative
- **FP (False Positive)** — ошибочно предсказано positive (на самом деле negative)
- **FN (False Negative)** — ошибочно предсказано negative (на самом деле positive)

Идеальная confusion matrix — диагональная (все числа на диагонали, нули вне).

## Метрики классификации

### Accuracy

```
Accuracy = (TP + TN) / (TP + TN + FP + FN)
```

Доля правильных ответов. Проста и интуитивна, но обманчива при несбалансированных
классах. Если 95% данных — класс A, модель, всегда отвечающая "A", имеет accuracy 95%.

### Precision

```
Precision = TP / (TP + FP)
```

"Из всех, кого модель назвала positive, сколько действительно positive?"
Высокий precision = мало ложных срабатываний.

### Recall

```
Recall = TP / (TP + FN)
```

"Из всех реально positive, сколько модель нашла?"
Высокий recall = мало пропущенных positive.

### F1-score

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

Гармоническое среднее precision и recall. F1 высокий только когда оба высоки.

### Macro F1

При нескольких классах — считаем F1 для каждого класса отдельно и берём среднее:

```
Macro F1 = (F1_negative + F1_positive) / 2
```

Каждый класс имеет равный вес, независимо от количества примеров.

## classification_report

`sklearn.metrics.classification_report` выводит все метрики сразу:

```
              precision    recall  f1-score   support
    negative       0.82      0.79      0.80       200
    positive       0.80      0.83      0.81       200
    accuracy                           0.81       400
   macro avg       0.81      0.81      0.81       400
```

- **support** — количество примеров этого класса в тестовой выборке

## Почему fine-tuning лучше (или не всегда)

Fine-tuning обычно даёт лучшие результаты, потому что модель подстраивает
attention-паттерны под задачу. Но есть случаи, когда baseline конкурентоспособен:

- **Мало данных** (<500 примеров) — fine-tuning может переобучиться
- **Короткие тексты + truncation** — если обрезка теряет важную информацию,
  обе модели страдают одинаково
- **Простая задача** — если тональность определяется одним словом ("terrible"),
  даже LogReg справляется

В нашем случае мы используем `max_length=64`, а IMDB-отзывы длинные (200+ слов).
Обрезка теряет часть контекста, что ограничивает обе модели.

## Подробный разбор кода

### predict_fine_tuned

```python
probs = torch.nn.functional.softmax(outputs.logits, dim=1)
pred = torch.argmax(probs, dim=1).item()
```

- `softmax(dim=1)` — нормализуем по оси классов (не по батчу)
- `.item()` — извлекает Python-число из тензора размера 1

### Confusion matrix с seaborn

```python
cm = confusion_matrix(test_labels, y_pred)
sns.heatmap(cm, annot=True, fmt="d", cmap="Blues")
```

- `annot=True` — подписать числа в ячейках
- `fmt="d"` — целые числа (не float)
- `cmap="Blues"` — цветовая палитра
