# День 5 — Fine-tuning модели

## Feature Extraction vs Fine-tuning

В День 4 мы использовали **feature extraction**: взяли замороженный DistilBERT,
извлекли эмбеддинги и обучили на них LogisticRegression. Получили F1 = 0.82.

Сегодня мы делаем **fine-tuning** — обучаем саму модель трансформера на нашей
задаче. Все веса модели (66 миллионов параметров) подстраиваются под
классификацию тональности.

```
Feature extraction (День 4):
  Текст → [DistilBERT замороженный] → CLS → [LogReg обучается] → класс
  Обучаемых параметров: ~768 (только LogReg)

Fine-tuning (День 5):
  Текст → [DistilBERT обучается] → [Классификатор обучается] → класс
  Обучаемых параметров: ~66 000 000 (вся модель)
```

Почему fine-tuning лучше? Когда все слои обучаются, модель подстраивает
свои attention-паттерны под задачу. Вместо того чтобы просто "понимать язык",
она начинает фокусироваться на тональных словах ("terrible", "amazing")
и игнорировать нерелевантные.

## AutoModelForSequenceClassification

В Дни 1–4 мы использовали `AutoModel` — он возвращает hidden states (вектора).
Сегодня мы используем `AutoModelForSequenceClassification` — он добавляет
**классификационную голову** поверх DistilBERT:

```
AutoModel:
  DistilBERT → last_hidden_state [batch, seq_len, 768]

AutoModelForSequenceClassification:
  DistilBERT → CLS [batch, 768] → Linear(768, num_labels) → logits [batch, 2]
```

Классификационная голова — это один линейный слой (`nn.Linear`), который
преобразует 768-мерный CLS-вектор в `num_labels` чисел (logits).

`num_labels=2` потому что у нас бинарная классификация: negative (0) и positive (1).

## PyTorch Dataset и DataLoader

### Dataset

PyTorch `Dataset` — абстрактный класс, от которого мы наследуемся.
Нужно определить два метода:

- `__len__()` — сколько примеров в датасете
- `__getitem__(idx)` — вернуть один пример по индексу

В `__getitem__` мы токенизируем текст и возвращаем словарь:

```python
{
    "input_ids": tensor([101, 2023, 3185, ...]),      # токены
    "attention_mask": tensor([1, 1, 1, ...]),           # маска
    "labels": tensor(1)                                 # метка класса
}
```

Важно: `padding="max_length"` — каждый пример паддится до одинаковой длины
(128 токенов). Это нужно, чтобы DataLoader мог собрать их в батч.
В отличие от `padding=True` (до максимума в батче), здесь мы фиксируем длину.

### DataLoader

DataLoader берёт Dataset и создаёт из него батчи:

```python
train_loader = DataLoader(train_dataset, batch_size=16, shuffle=True)
```

- `batch_size=16` — по 16 текстов за раз (ограничение памяти)
- `shuffle=True` — перемешивать данные каждую эпоху (для train)
- Для val_loader `shuffle=False` — порядок не важен при оценке

DataLoader автоматически собирает отдельные примеры в батч:
16 тензоров `[128]` → один тензор `[16, 128]`.

## Обучение: эпохи, loss, градиенты

### Эпоха

Одна **эпоха** — это один полный проход по всему обучающему датасету.

При 1600 примерах и batch_size=16:
- За эпоху будет 1600/16 = 100 шагов (итераций)
- На каждом шаге обрабатывается один батч из 16 текстов

### Что происходит на каждом шаге

```python
optimizer.zero_grad()     # 1. Обнуляем градиенты
outputs = model(...)      # 2. Forward pass: текст → предсказание
loss = outputs.loss       # 3. Считаем ошибку (Cross-Entropy Loss)
loss.backward()           # 4. Backward pass: вычисляем градиенты
optimizer.step()          # 5. Обновляем веса модели
```

**Forward pass** — данные проходят через модель "вперёд": от input_ids до logits.

**Loss (функция потерь)** — Cross-Entropy Loss, стандарт для классификации.
Она измеряет, насколько предсказанные вероятности отличаются от правильных меток.
Если модель уверена в правильном ответе — loss маленький, если ошибается — большой.

**Backward pass** — алгоритм backpropagation вычисляет градиенты: "в какую сторону
и насколько нужно сдвинуть каждый из 66M параметров, чтобы loss уменьшился".

**optimizer.step()** — AdamW сдвигает каждый параметр в направлении,
уменьшающем loss. Learning rate (2e-5 = 0.00002) контролирует размер шага.

### Почему lr = 2e-5?

Для fine-tuning используют очень маленький learning rate:

| Задача | Типичный LR |
|--------|------------|
| Обучение с нуля | 1e-3 — 1e-1 |
| Fine-tuning трансформера | 1e-5 — 5e-5 |
| Обучение только головы | 1e-3 — 1e-2 |

Маленький LR нужен, потому что предобученные веса уже хорошие — мы хотим
их слегка подстроить, а не сломать. Слишком большой LR приведёт к
**catastrophic forgetting** — модель забудет всё, чему научилась при pre-training.

## AdamW

AdamW — это оптимизатор, стандартный для fine-tuning трансформеров:

- **Adam** — адаптивный оптимизатор, который подбирает свой learning rate
  для каждого параметра на основе истории градиентов
- **W (Weight Decay)** — регуляризация, которая слегка "уменьшает" веса
  на каждом шаге, предотвращая переобучение

Мы используем `torch.optim.AdamW` вместо устаревшего `transformers.AdamW`.

## Функция оценки (evaluate)

```python
model.eval()          # отключает dropout
with torch.no_grad(): # отключает градиенты — экономит память
```

`outputs.logits` — "сырые" предсказания модели (не нормализованные).
Для двух классов это тензор `[batch, 2]`:

```
logits = [[−1.5, 2.3],   # второй класс (positive) выше → positive
          [3.1, −0.8]]    # первый класс (negative) выше → negative
```

`torch.argmax(logits, dim=1)` — берём индекс максимального значения:
`[1, 0]` → [positive, negative].

## model.train() vs model.eval()

| | model.train() | model.eval() |
|---|--------------|-------------|
| Dropout | Включён (случайно обнуляет нейроны) | Выключен |
| BatchNorm | Использует статистику батча | Использует накопленную статистику |
| Когда | При обучении | При оценке и инференсе |

Dropout — это метод регуляризации: при обучении случайные 10% нейронов
отключаются на каждом шаге. Это заставляет модель не полагаться на
отдельные нейроны и улучшает обобщение. При оценке dropout не нужен —
мы хотим стабильный результат.

## Сохранение модели

```python
model.save_pretrained("./fine_tuned_model")
tokenizer.save_pretrained("./fine_tuned_model")
```

Сохраняет:
- `config.json` — архитектура модели (число слоёв, размерности, num_labels)
- `model.safetensors` — обученные веса (~260 MB)
- `tokenizer.json`, `vocab.txt` — словарь и настройки токенизатора

Потом можно загрузить:
```python
model = AutoModelForSequenceClassification.from_pretrained("./fine_tuned_model")
tokenizer = AutoTokenizer.from_pretrained("./fine_tuned_model")
```

## Ожидаемый результат

С fine-tuning на 2000 IMDB отзывах за 3 эпохи:
- Baseline (День 4): F1 ≈ 0.82
- Fine-tuned (День 5): F1 ≈ 0.87–0.92

Разница возникает потому, что теперь DistilBERT учится фокусировать attention
на тональных словах, а не просто "понимать английский".
