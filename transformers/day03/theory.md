# День 3 — Attention-матрицы и визуализация

## Что такое attention weights?

В Дне 1 мы разобрали формулу self-attention:

```
Attention(Q, K, V) = softmax(Q · Kᵀ / √d) · V
```

**Attention weights** — это результат `softmax(Q · Kᵀ / √d)`, то есть матрица
весов **до** умножения на Value. Эта матрица показывает, насколько каждый токен
"обращает внимание" на каждый другой.

```
                 The  amazing  movie  won  many  awards
The            [ 0.3    0.1    0.2   0.1   0.1   0.2  ]
amazing        [ 0.1    0.2    0.4   0.1   0.1   0.1  ]
movie          [ 0.1    0.3    0.2   0.2   0.1   0.1  ]
won            [ 0.1    0.1    0.3   0.2   0.1   0.2  ]
many           [ 0.1    0.1    0.1   0.1   0.3   0.3  ]
awards         [ 0.1    0.1    0.1   0.2   0.3   0.2  ]
```

Каждая строка — это **Query** (кто смотрит), каждый столбец — **Key** (на кого смотрят).
Значения в строке всегда дают сумму 1 (результат softmax). Чем выше значение,
тем больше внимания токен-запрос уделяет токену-ключу.

В примере выше: "amazing" больше всего смотрит на "movie" (0.4) — логично,
прилагательное связано с существительным, которое оно описывает.

## Форма attention тензора

```python
outputs.attentions[0].shape  # torch.Size([1, 12, 8, 8])
                                           │  │   │  │
                                           │  │   │  └─ seq_len (ключи)
                                           │  │   └─ seq_len (запросы)
                                           │  └─ num_heads (12 голов)
                                           └─ batch_size (1 текст)
```

- `outputs.attentions` — кортеж из 6 элементов (по одному на слой DistilBERT)
- `outputs.attentions[0]` — attention первого слоя
- Каждый слой: `[batch, heads, seq_len, seq_len]`
- У DistilBERT 12 голов, поэтому `num_heads = 12`

### Почему output_attentions=True?

По умолчанию модель не возвращает attention weights — они нужны только для
анализа и отладки. Хранение attention для 6 слоёв × 12 голов × seq_len² занимает
много памяти, поэтому его нужно запросить явно:

```python
model = AutoModel.from_pretrained(model_name, output_attentions=True)
```

## Multi-Head Attention: что видит каждая голова

В DistilBERT 12 голов attention на каждом слое. Каждая голова работает в своём
подпространстве размерности 64 (768 / 12 = 64) и может выучить свой тип связи.

Исследования (Clark et al., 2019 — "What Does BERT Look At?") показали, что
головы специализируются:

| Тип паттерна | Что делает | Пример |
|-------------|-----------|--------|
| **Diagonal** | Токен обращает внимание на самого себя | Самосвязь |
| **Previous token** | Внимание на предыдущий токен | Языковая модель слева направо |
| **Next token** | Внимание на следующий токен | Обратный контекст |
| **[SEP] attention** | Все токены смотрят на [SEP] | [SEP] как "мусорная корзина" |
| **[CLS] attention** | Все токены смотрят на [CLS] | Агрегация информации |
| **Syntactic** | Связь подлежащее ↔ сказуемое | "cat" → "sat" |
| **Coreference** | Местоимение → антецедент | "he" → "John" |
| **Positional band** | Внимание на токены рядом (±2-3 позиции) | Локальный контекст |

Когда мы визуализируем разные головы одного слоя, мы видим эти паттерны.

### Паттерн "[SEP] как мусорная корзина"

Интересное наблюдение: многие головы направляют большую часть внимания на `[SEP]`.
Это происходит потому, что когда токену "нечего сказать" (нет значимой связи),
модель учится направлять внимание на `[SEP]` — безопасный токен, который не
изменит представление. Это аналог "no-op" (пустой операции).

## Как attention меняется по слоям

### Нижние слои (0–1): локальные паттерны

На первых слоях attention обычно сфокусирован на:
- Соседних токенах (±1-2 позиции)
- Самом себе (диагональ матрицы)
- Специальных токенах [CLS] и [SEP]

Это логично: модель сначала обрабатывает локальный контекст — какие слова
стоят рядом, какая часть речи у текущего токена.

### Средние слои (2–3): синтаксические связи

На средних слоях появляются более длинные связи:
- Подлежащее ↔ сказуемое (даже через несколько слов)
- Прилагательное → существительное
- Предлог → его аргументы

### Верхние слои (4–5): семантические паттерны

На последних слоях:
- Attention более "размытый" — токены обращают внимание на многие позиции
- Важные для задачи слова получают больше внимания
- [CLS]-токен активно собирает информацию со всех позиций

Именно поэтому вектор [CLS] из последнего слоя используется для классификации —
к этому моменту он "увидел" весь текст через attention.

## Heatmap: как читать визуализацию

```
            [CLS]  The  amazing  movie  won  many  awards  [SEP]
[CLS]       ████   ██    ██      ██    ██    ██     ██     ██
The          ██   ████    █       █     ·     ·      ·      █
amazing      ██    ██   ████    ███     ·     ·      ·      █
movie        ██    ██   ████   ████    ██     ·      ·      █
won          ██     ·    ██     ██   ████    ██     ██      █
many         ██     ·     ·      ·    ██   ████   ████      █
awards       ██     ·     ·      ·    ██   ████   ████      █
[SEP]        ██    ██    ██     ██    ██    ██     ██     ████
```

- Яркая ячейка (█) = высокий вес attention = сильная связь
- Тёмная ячейка (·) = низкий вес = слабая связь
- Строка = Query (кто смотрит)
- Столбец = Key (на кого смотрят)
- Диагональ обычно яркая — токен всегда обращает внимание на себя

### Цветовая схема viridis

Мы используем `cmap='viridis'` — палитру от тёмно-фиолетового (0) через
зелёный (0.5) до жёлтого (1). Она выбрана потому что:
- Перцептивно равномерная (одинаковые разницы в данных → одинаковые различия в цвете)
- Хорошо видна при чёрно-белой печати
- Доступна для людей с дальтонизмом

## Среднее attention по головам

Иногда полезно посмотреть на **среднее** attention по всем головам:

```python
avg_attention = outputs.attentions[5][0].mean(dim=0)  # [seq_len, seq_len]
```

Это усредняет по оси `num_heads`, показывая общую картину того, на что модель
обращает внимание. Но будьте осторожны: усреднение может скрыть интересные
паттерны отдельных голов (например, голова, отвечающая за синтаксис, будет
"размыта" другими головами).

## Attention ≠ объяснение

Важное предупреждение: attention weights показывают, как модель перемешивает
информацию между позициями, но это **не то же самое, что объяснение предсказания**.

Почему attention не равен важности:
1. **Множество слоёв** — информация проходит через 6 слоёв, и attention
   на одном слое не учитывает трансформации на других
2. **Residual connections** — к выходу attention добавляется исходный вход,
   поэтому даже с нулевым attention токен сохраняет свою информацию
3. **FFN слои** — между слоями attention стоят FFN, которые сильно
   трансформируют представления
4. **Attention rollout** — для корректного анализа нужно учитывать
   **все** слои attention и residual connections (метод "attention rollout")

Тем не менее, визуализация attention полезна для:
- Понимания структуры модели
- Отладки (модель не смотрит на нужные слова?)
- Научных исследований
- Построения интуиции о трансформерах

## Практические паттерны attention

### "Amazing movie" — прилагательное + существительное

На ранних слоях "amazing" обычно сильно обращает внимание на "movie".
Это потому что модель выучила, что прилагательные модифицируют ближайшее
существительное, и нужно объединить их значения.

### "Terrible... hated" — тональность

На поздних слоях в предложении "This movie was absolutely terrible and I hated it"
слова "terrible" и "hated" обычно обращают внимание друг на друга.
Модель улавливает, что оба слова несут негативную тональность и усиливают
друг друга.

### Внимание к функциональным словам

Интересно, что "was", "and", "it" обычно получают мало attention на поздних
слоях — модель "понимает", что они несут мало семантической информации,
и фокусируется на содержательных словах.

## Альтернативные методы визуализации attention

### BertViz

Библиотека `bertviz` предоставляет интерактивную визуализацию attention
в Jupyter notebooks — можно навести мышку на токен и увидеть, на что он смотрит.

### Attention Rollout

Метод, учитывающий все слои + residual connections:

```
R₀ = I (единичная матрица)
Rₗ = 0.5 × Aₗ × Rₗ₋₁ + 0.5 × Rₗ₋₁
```

где `Aₗ` — attention матрица слоя l, `Rₗ` — "суммарный" attention до слоя l.
Множитель 0.5 учитывает residual connection (половина информации идёт напрямую).

### Gradient-based методы

Более надёжные методы объяснения (Integrated Gradients, SHAP) используют
градиенты модели вместо attention weights. Они показывают, какие входные
токены реально повлияли на предсказание, а не просто куда "смотрит" модель.
